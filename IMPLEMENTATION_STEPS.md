# Покутська пекарня: кроки впровадження

## 1. Головна сторінка

1. Логотип додано в `assets/logo.svg`, а оригінальний PDF збережено в `assets/PokytskaPekarnya_LOGO.pdf`.
2. Основні кольори винесені в `:root` файлу `style.css`: бордовий з логотипа, темний графіт, теплий кремовий фон і пшеничні акценти.
3. У `index.html` змінено слоган на `Традиції рідного краю`.
4. Заголовок `ПОКУТСЬКА ПЕКАРНЯ` зменшено через `.hero__title`.
5. Ціни прибрано з усіх карток товарів.
6. Форму зворотного зв'язку видалено, у блоці контактів залишено адресу, телефон, email, графік, соцмережі та карту.
7. Поруч із заголовком додано блок `Новинки`; текст підтягується з `/api/settings/news`.

## 2. Сторінки категорій

1. Картки категорій на головній тепер ведуть на `category.html?category=bread`, `category.html?category=buns` тощо.
2. `category.js` читає параметр `category` з URL.
3. Якщо сайт відкрито через сервер, товари завантажуються з `/api/products?category=...`.
4. Якщо файл відкрито напряму без сервера, показується демо-набір товарів.

## 3. Адмін-панель і база

1. Адмінка доступна за адресою `/admin.html`.
2. Дані зберігаються в `data/db.json`.
3. Фото нових товарів завантажуються в папку `imagines/uploads/`.
4. Ендпоінти:
   - `GET /api/settings/news` повертає текст блоку “Новинки”.
   - `PUT /api/settings/news` оновлює текст блоку “Новинки”. Body: `{ "value": "..." }`.
   - `GET /api/products` повертає всі товари.
   - `GET /api/products?category=bread` повертає товари однієї категорії.
   - `POST /api/products` додає товар через `multipart/form-data` з полями `name`, `category`, `description`, `image`.

## 4. Запуск

1. Відкрийте папку проєкту в терміналі.
2. Запустіть сервер:

```bash
npm start
```

3. Відкрийте сайт:

```text
http://localhost:3000
```

4. Відкрийте адмінку:

```text
http://localhost:3000/admin.html
```

## 5. Структура бази даних

```json
{
  "settings": {
    "news": "Текст інформаційного блоку"
  },
  "products": [
    {
      "id": "bread-sourdough",
      "name": "Хліб на заквасці",
      "category": "bread",
      "description": "Опис товару",
      "image": "imagines/sourdough.png",
      "createdAt": "2026-05-10T00:00:00.000Z"
    }
  ]
}
```

Для реального продакшену варто додати авторизацію адміністратора і перенести `data/db.json` у SQLite або PostgreSQL, але поточна структура API вже підготовлена для такого переходу.
